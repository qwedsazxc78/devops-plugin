## Severity Definitions

| Severity | Definition | SLA |
|----------|-----------|-----|
| **Critical** | Immediate exploitation risk. Data exposure, cluster compromise, or privilege escalation possible now. | Fix within 24 hours |
| **High** | Significant security gap. Exploitation requires additional conditions but impact would be severe. | Fix within 1 week |
| **Medium** | Defense-in-depth gap. Not immediately exploitable but weakens overall security posture. | Fix within 1 month |
| **Low** | Best practice deviation. Minimal direct security impact but should be addressed for hygiene. | Fix within 1 quarter |
| **Info** | Informational. Configuration is acceptable but could be improved, or notable for awareness. | No SLA |

## Category Definitions

| Category | Scope |
|----------|-------|
| **GKE Hardening** | Cluster-level settings in GKE configuration files (discovered at runtime) |
| **IAM** | Service accounts, roles, bindings in identity configuration files (discovered at runtime) |
| **Secrets** | Hardcoded credentials, secret management practices |
| **Network** | Network policies, firewall rules, endpoint exposure |
| **Helm Security** | Per-chart RBAC, service accounts, resource limits |
| **Compliance** | CIS benchmark alignment, regulatory requirements |

## Quick Scan Format

```markdown
## Quick Security Scan

**Critical:** N | **High:** N | **Medium:** N

### Critical Findings
- [CRIT-1] [One-line description] → [File] → [Fix]

### High Findings
- [HIGH-1] [One-line description] → [File] → [Fix]

### Action Required
[Yes/No — are there any blocking findings?]
```
